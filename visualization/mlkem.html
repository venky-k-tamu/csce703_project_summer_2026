Module-LWE structure

The lattice problem

The public key hides the secret ŝ behind a Module-LWE sample: t̂ = Â ∘ ŝ + ê, where Â is a k × k = 3 × 3 matrix of polynomials, and ŝ, ê are length-k vectors of short (small-coefficient) polynomials. Recovering ŝ from (Â, t̂) is the hard problem.

Everything is a polynomial

Every entry lives in the ring R_q = ℤ_q[X]/(X²⁵⁶ + 1), q = 3329 — a degree-255 polynomial with 256 coefficients. The matrix Â and vectors are stored in the NTT domain (hats), so the matrix–vector product uses cheap pointwise multiplication ∘.

Matrix indexing

Â[i][j] = SampleNTT(ρ ‖ j ‖ i) — byte order (j, i), i.e. (col, row) — per FIPS 203 Alg 13. KeyGen multiplies by Â; K-PKE encryption uses Âᵀ.

Number-Theoretic Transform — butterfly network
Demo shows an 8-point radix-2 network (3 stages); ML-KEM runs 7 of 8 stages on 256 points

NTT — fast polynomial multiplication

Why the NTT

Multiplying two degree-255 polynomials naively is O(n²). The NTT maps a polynomial to an evaluation basis where multiplication becomes pointwise, cutting a product to O(n log n).

The ring splits into 128 quadratics

ζ = 17 is a primitive 256th root of unity mod 3329 (not a 512th), so X²⁵⁶ + 1 factors into 128 quadratic factors X² − ζ2·BitRev₇(i)+1, not 256 linear ones. The NTT therefore runs 7 of 8 butterfly stages and stops at 128 degree-1 pairs.

Pointwise multiply

Because each factor is a quadratic, pointwise multiplication is base_case_multiply — a product of two degree-1 polynomials mod (X² − γ), not a single scalar product. ZETAS[k] = ζBitRev₇(k) mod q.

Sampling into R_q

SampleNTT — rejection sampling

Â is uniform mod q. A SHAKE-128 XOF stream is parsed 3 bytes at a time into two 12-bit values d₁, d₂. Each is accepted iff < q = 3329, else rejected — this rejection makes the output uniform over ℤ_q. Acceptance rate ≈ 3329/4096 ≈ 81.3%.

SamplePolyCBD — centered binomial

The secret ŝ and errors are short: each coefficient is drawn from a centered binomial distribution CBDη. Take 2η random bits, split into a and b, and output popcount(a) − popcount(b) ∈ [−η, η].

Distribution is symmetric and concentrated at 0 — that "shortness" is what keeps decryption correct.

Compress_d / Decompress_d — lossy rounding
Compression discards low bits: ℤ_q (3329 values) → ℤ_{2ᵈ} buckets and back

Compression

The rounding map

Ciphertexts are shrunk by dropping low-order bits. Compress_d(x) = round(2ᵈ/q · x) mod 2ᵈ quantizes ℤ_q into 2ᵈ buckets; Decompress_d(y) = round(q/2ᵈ · y) maps back to a representative. The round-trip error is bounded by ≈ q/2^{d+1}.

Where it is used

d = 1 encodes the message bit (0 → 0, 1 → ⌈q/2⌉ = 1665) — the widest gap, so noise up to ~q/4 still decodes. d = d_u, d_v compress the ciphertext parts u and v.
